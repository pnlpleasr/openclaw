External Tool Verifier

A pre-execution verification gateway that consults external authorities — webhooks, LLMs, or humans — before any tool call proceeds.

Context

Why This Exists

The Gap

  • Existing layers control what can run and where
  • No way to consult an external authority before execution
  • Autonomous agents run unattended with no human oversight
  • No integration point for enterprise policy servers

What This Adds

  • Route tool calls to another LLM for safety analysis
  • Human-in-the-loop approval via Telegram (or other channels)
  • Automated policy evaluation via webhook endpoints
  • Plugs into existing pipeline — purely additive, no breaking changes
Architecture

Where It Fits

The verifier is Layer 3 in OpenClaw's security pipeline. It only sees tool calls that already passed existing checks.

Telegram Approval

Sends an inline-keyboard message with Allow/Deny buttons. Human taps to approve. Uses direct API calls (no long-polling) to avoid race conditions.

Behavior

Decision Matrix

What happens under every condition, for each failMode setting.

Condition failMode: "deny" failMode: "allow"
Webhook returns "allow" Allowed Allowed
Webhook returns "deny" Blocked Blocked
Webhook timeout / unreachable Blocked Allowed
Non-2xx HTTP status Blocked Allowed
Telegram — no response within timeout Blocked Allowed
Telegram — unauthorized user taps button Ignored (alert shown to user)

Per-agent failMode uses most-restrictive-wins: agent "allow" + global "deny" → effective "deny".

Security Review

Hardening Measures

Identified during tech lead review. All P0/P1 issues addressed in the plan.

R

Param Redaction

File contents stripped from write/edit params before sending to external endpoints. Prevents secret leakage.

H

HTTPS Enforced

Webhook URLs must use HTTPS in production. HTTP rejected at config validation time.

A

Sender Validation

allowedUserIds restricts who can tap Allow/Deny. Unauthorized taps rejected with alert.

P

No Long-Polling

Telegram uses direct API calls. Avoids 409 conflicts, race conditions, and lost callbacks.

F

Most-Restrictive-Wins

Per-agent failMode cannot weaken global setting. Prevents misconfigured agents from degrading security.

L

Response Limits

64KB body cap, 500-char reason truncation. Prevents memory abuse from malicious endpoints.

Configuration

Quick Start

Add to openclaw.json — that's it. Works with existing tool policy and sandbox settings.

// openclaw.json { "tools": { "verifier": { "enabled": true, "failMode": "deny", "scope": { "include": ["exec", "write", "edit"] }, // Route to an LLM safety checker, policy server, or custom endpoint "webhook": { "url": "https://my-verifier.example.com/verify", "timeout": 30, "secret": "${VERIFIER_HMAC_SECRET}" }, // Human-in-the-loop via Telegram "telegram": { "enabled": true, "botToken": "${TELEGRAM_BOT_TOKEN}", "chatId": "123456789", "allowedUserIds": [123456789] } } } }
